- Entry `src/main.jsx` mounts the React 18 root via `createRoot`, wraps everything in `<StrictMode>`, and conditionally registers the service worker at `/sw.js`.
- `src/App.jsx` composes providers: `I18nProvider` from `lib/I18nContext.jsx` around `react-router-dom`'s `BrowserRouter` + `Routes`; page transitions are driven by Framer Motion's `AnimatePresence` keyed on `location.pathname`.
- Two routes — `/` → `HomePage` (the main interview-coach flow) and `/settings` → `SettingsPage` — with a catch-all redirect to `/`. Every page is wrapped in `components/Shell.jsx`, which renders the brand header, nav chips, language switcher, and an `InstallPrompt` for PWA installability.
- `src/lib/` holds pure utilities consumed by pages/components: `i18n.js` + `I18nContext.jsx` provide UI-language context; `settingsConfig.js` normalizes/persists settings and resolves API keys (user-pasted key wins over build-time `VITE_GEMINI_API_KEY`); `prompt.js` builds system/user prompts; `candidate.js` / `jobMeta.js` extract metadata; `tts.js` wraps Web Speech API; `exportPdf.js` drives PDF generation; `storage.js` wraps `localStorage` with JSON serialization; `mandatoryQuestions.js`, `interviewModes.js`, `ocr.js`, `fetchUrl.js` are feature helpers.
- `public/sw.js` implements a cache-first strategy for static assets while bypassing `/api/*` so server-side key checks and TTS stay live; `public/manifest.json` declares the PWA (standalone display, maskable icons, portrait-primary orientation). The SPA has no client-side router history persistence — navigation is shallow URL-based.
