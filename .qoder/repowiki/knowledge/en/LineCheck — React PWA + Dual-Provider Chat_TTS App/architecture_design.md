Monorepo-style single project split into three runtime layers:

- **Vite + React SPA** (`src/`): `main.jsx` bootstraps a `BrowserRouter` wrapping an `AnimatedRoutes` component (framer-motion page transitions) and an `I18nProvider`. Pages live in `src/pages/`, shared UI in `src/components/`, and pure utilities in `src/lib/` (i18n, PDF export, OCR, TTS client). Tailwind v4 is configured via `@tailwindcss/vite`; `public/sw.js` registers a service worker for offline/PWA support.
- **Vercel serverless API** (`api/`): Each file exports a Next/Vercel-compatible handler plus a `config.maxDuration` envelope. `api/chat.js` is the core: it parses `x-linecheck-ai-*` headers to pick between Google Gemini (`generativelanguage.googleapis.com/v1beta/openai`) and DeepSeek (`api.deepseek.com`), rotates through a pool of keys collected from `GEMINI_API_KEY[_N]` / `DEEPSEEK_API_KEY` env vars, enforces a 55s handler budget with per-call 50s timeouts, and implements failover across providers and keys. `api/tts.js` and `api/fetch-url.js` proxy to the Python TTS server; `api/_ttsShared.js` centralises CORS helpers.
- **Local dev stack**: `scripts/dev-api-server.mjs` is a standalone Node HTTP server mirroring `api/chat.js` logic on port 8791 so the Vite dev server can proxy `/api/*` without Vercel. A separate Python process (`scripts/tts_server.py`, port 8790) serves TTS endpoints proxied by Vite's `vite.config.js` rewrite rules. The root `npm run dev` orchestrates all three processes concurrently.

Dependency direction is one-way: browser code calls `/api/*` routes; serverless handlers call upstream LLM APIs or the local Python TTS server. There are no cross-imports between `src/` and `api/`.
