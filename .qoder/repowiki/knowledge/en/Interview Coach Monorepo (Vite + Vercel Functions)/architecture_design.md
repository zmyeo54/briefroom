Three sibling concerns are glued together by a single runtime contract — every browser request goes to `/api/<service>` paths. In development `vite.config.js` proxies `/api/chat`, `/api/tts*`, and `/api/fetch-url` to the Node OpenAI-compatible proxy (`scripts/dev-api-server.mjs`) on :8791 and the Python Edge-TTS server (`scripts/tts_server.py`) on :8790; in production `vercel.json` declares the same three handlers as Vercel Functions with per-route `maxDuration`, while all other routes fall back to `index.html`. The root `package.json` is the only entry point: `npm run dev` concurrently starts the TTS server, the API proxy, and the Vite dev server (:8787), and `npm run build` emits a Vercel-ready `dist/`. Shared configuration (API base URLs, Gemini keys, TTS settings) flows via environment variables consumed uniformly by both the frontend and the serverless routes, so no child hard-codes upstream hostnames.
