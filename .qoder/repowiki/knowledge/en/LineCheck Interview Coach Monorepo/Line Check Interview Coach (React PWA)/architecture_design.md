- Entry point `src/main.jsx` mounts the React 18 root via `createRoot`, wraps everything in `<StrictMode>` and registers the service worker at `/sw.js`.
- `src/App.jsx` composes the top-level providers (`I18nProvider` from `lib/I18nContext.jsx`) around `react-router-dom`'s `BrowserRouter` + `Routes`, with Framer Motion's `AnimatePresence` driving page transitions keyed by `location.pathname`.
- Two routes: `/` → `HomePage` (the main interview-coach flow) and `/settings` → `SettingsPage`; a catch-all route redirects to `/`.
- `src/components/Shell.jsx` is the shared chrome wrapping every page — it renders the brand header, nav chips, language switcher, and an `InstallPrompt` for PWA installability.
- `src/lib/` holds pure utilities consumed by pages/components: `i18n.js` + `I18nContext.jsx` provide UI-language context; `settingsConfig.js` normalizes/persists settings; `prompt.js` builds system/user prompts; `candidate.js` / `jobMeta.js` extract metadata; `tts.js` wraps Web Speech API; `exportPdf.js` drives PDF generation; `storage.js` wraps `localStorage` with JSON serialization; `mandatoryQuestions.js`, `interviewModes.js`, `ocr.js`, `fetchUrl.js`, `generateLoading.jsx` are feature helpers.
- `public/manifest.json` declares the PWA (standalone display, maskable icons, portrait-primary orientation); `public/sw.js` implements a cache-first strategy for static assets while bypassing `/api/*` so server-side key checks and TTS stay live. The SPA has no client-side router history persistence — navigation is shallow URL-based.
