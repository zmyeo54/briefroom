The repository is a flat workspace that composes three sibling concerns around one contract: the browser app talks exclusively to `/api/*` endpoints, which are served by Vite's dev proxy in development (vite.config.js forwards `/api/chat`, `/api/tts*`, `/api/fetch-url` to `scripts/dev-api-server.mjs` on ports 8791/8790) and by Vercel Functions in production (vercel.json declares `api/chat.js`, `api/tts.js`, `api/fetch-url.js` as functions with explicit maxDuration). The root `package.json` orchestrates the full local stack via the `dev` script, which concurrently starts the Python Edge-TTS server (`scripts/tts_server.py`), the Node OpenAI-compatible proxy (`scripts/dev-api-server.mjs`), and the Vite dev server on port 8787. Production builds go through `vite build` into `dist/`, which Vercel serves as static assets while routing non-API paths back to `index.html`. Shared runtime configuration (API base URLs, Gemini keys, TTS settings) flows from environment variables consumed uniformly by both the frontend and the serverless routes.
