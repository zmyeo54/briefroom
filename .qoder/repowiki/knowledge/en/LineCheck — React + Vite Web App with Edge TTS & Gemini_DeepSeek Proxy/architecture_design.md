Monorepo-style root containing three layers:
- Frontend (`src/`): React 19 + Vite 8 entry at `src/main.jsx`, routing via `react-router-dom` in `src/App.jsx` (two routes: `/` → `HomePage`, `/settings` → `SettingsPage`). UI is split into presentational components under `src/components/` and page-level containers under `src/pages/`. Shared client logic lives in `src/lib/` (i18n, storage, OCR, PDF export, prompt building, TTS).
- API layer (`api/`): Vercel Serverless Functions (Node.js) — each file exports a default handler plus a `config` object (e.g. `maxDuration`, `bodyParser.sizeLimit`). `api/chat.js` is the core proxy that selects between Google Gemini (`generativelanguage.googleapis.com/v1beta/openai`) and DeepSeek (`api.deepseek.com`) based on headers `x-linecheck-ai-region`, `x-linecheck-ai-provider`, `x-linecheck-ai-enabled`, and falls back across multiple keys collected from `GEMINI_API_KEY[_N]` / `VITE_GEMINI_API_KEY` / `DEEPSEEK_API_KEY` env vars. `api/_ttsShared.js` supplies shared helpers (CORS). Other endpoints (`tts.js`, `tts-health.js`, `fetch-url.js`) wrap the Python TTS service.
- Dev tooling: `scripts/dev-api-server.mjs` is a standalone Node `http` server mirroring `api/chat.js`'s provider-selection logic for local development; `vite.config.js` proxies `/api/*` to it (chat→8791, tts/tts-health/fetch-url→8790). `scripts/tts_server.py` runs the actual TTS backend (Python, invoked via `.venv/bin/python`).
- Build/deploy: `vercel.json` declares the Vercel deployment; static assets live in `public/` (PWA manifest, icons, SW); a legacy flat copy exists in `_legacy_flat/` for reference only.
